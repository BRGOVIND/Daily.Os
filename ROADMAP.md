# Roadmap

Daily OS grows slowly and deliberately. Features are added only when they
strengthen the calm, calendar-native core — never to chase a checklist.

## Shipped

- **Phase 1 — Foundations**: calendar homepage, full-day workspace, tasks,
  Today's Focus, notes, offline storage (IndexedDB).
- **Phase 2 — Productivity**: templates, habits with streaks, recurring tasks,
  reminders, search, filtering, daily review, plan-tomorrow, statistics.
- **Phase 3 — Refinement**: unified motion language, welcoming empty states,
  the Growth Tree journal, milestone celebrations, landing page, PWA install
  + offline caching, and open-source polish.

## Considering

- [ ] Optional dark theme (the token layer is already theme-ready).
- [ ] Real reminder scheduling that survives the app being closed (via the
      service worker / Notification triggers where supported).
- [ ] Rich-text or checklist notes.
- [ ] Sub-tasks / checklists within a task.
- [ ] Weekly & monthly review flows, complementing the daily review.
- [ ] Encrypted export and optional end-to-end-encrypted sync.
- [ ] PNG/maskable icon set and screenshots in the manifest for richer installs.
- [ ] Localization / i18n.

## Explicitly out of scope

To protect the product's calm identity, Daily OS will **not** add: accounts or
login, a required backend, AI assistants, chat, Kanban boards, a left sidebar,
project hierarchies, or analytics/tracking.

Have an idea? Open a
[feature request](.github/ISSUE_TEMPLATE/feature_request.md) and tell us how it
keeps the spirit.
